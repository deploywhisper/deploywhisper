# DeployWhisper v1.2.0

DeployWhisper v1.2.0 is the new UI migration release. It promotes the React
SPA to the primary web interface, serves it from FastAPI at root routes, removes
the retired Python UI package from the runtime, and keeps the deterministic
analysis core and advisory-first posture intact.

## Highlights

- React is now the root web UI at `/`; legacy `/app/...` links redirect to the
  corresponding root SPA routes.
- The retired NiceGUI UI package, UI tests, static assets, and runtime
  dependency path were removed from the application package.
- The production image now builds the frontend in a Node stage and ships only
  compiled SPA assets in the Python runtime.
- The application version now reports `1.2.0` from runtime API metadata and
  package metadata.

## New React UI

- Added the React dashboard with project-aware KPI cards, recent analyses, latest
  briefing, upload workflow, verdict health, and persistent project selection.
- Added the React report experience with sticky verdict header, Overview,
  Findings, Confidence, Context, Rollback, and Audit tabs, public shared-report
  support, copy-briefing toast, reviewer feedback, and narrative-degraded
  handling.
- Added the React history screen with server-side search, severity and verdict
  filters, pagination, expanded row details, rescan-diff display, and destructive
  bulk-delete confirmation.
- Added React settings, incidents, and skills screens covering provider settings,
  topology/custom-skill controls, incident ingestion, and skill catalog browsing.
- Added the DeployWhisper logo and favicon assets to the frontend shell.
- Improved the report blast-radius card with a visual radius treatment and a
  positive safe state when blast radius is zero.
- Fixed dashboard verdict-health counting so `GO` verdicts are counted as
  `Clear`.
- Fixed dashboard greeting copy so it changes by local time of day.
- Fixed selected-project persistence across refreshes and page navigation.
- Added success messaging for settings updates.

## Backend and API

- Added `GET /api/v1/stats/summary` for dashboard KPI totals and seven-bucket
  sparkline series.
- Added `GET /api/v1/stats/verdict-distribution?days=30` for verdict health.
- Kept `GET /api/v1/projects` as the project list endpoint and added `name` and
  `env_label` aliases for the React project switcher.
- Extended `GET /api/v1/analyses` additively with dashboard/report fields such
  as severity, verdict/recommendation, score, environment/workspace labels,
  duration, filenames, PR refs, and trigger refs.
- Added a backend-for-UI bulk delete wrapper for analysis history.
- Added settings endpoints and additive skills API metadata for the Phase 6 React
  screens.
- Regenerated the frontend OpenAPI schema and typed API client.

## Local AI and Developer Workflow

- Updated local Ollama defaults for development to use
  `ollama/qwen2.5-coder:3b`, targeting faster local narrative testing for code
  and deployment-review tasks.
- Documented that UI E2E, accessibility, keyboard, and screenshot checks should
  run against the Docker Compose app at `http://localhost:8080/`, not a Vite dev
  server.
- Added React frontend workspace scripts for typecheck, Vitest, production build,
  OpenAPI generation, and Playwright checks.
- Added frontend CI coverage for typecheck, tests, and production build.
