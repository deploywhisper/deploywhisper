# DeployWhisper v1.1.0

DeployWhisper v1.1.0 builds on the v1.0.0 launch by turning the advisory workflow into a stronger integration, benchmarking, topology, and ownership-mapping platform. The release covers the work merged after v1.0.0 through Story 7.4, including GitHub Action reporting, benchmark quality gates, calibration history, topology connectors, Kubernetes live-state context, and CODEOWNERS-based ownership mapping.

## Compared With v1.0.0

v1.0.0 established the core product foundation: evidence-backed deployment reviews, project/workspace scoping, report history, incident memory, feedback/outcome learning foundations, topology context, skills marketplace support, and GitHub/self-hosted delivery.

v1.1.0 focuses on making those foundations easier to automate and trust in real delivery workflows:

- API and CLI consumers now have a versioned report contract and project-aware advisory output.
- GitHub Action users get stronger integration contracts, PR comments, rerun comparison behavior, and future adapter compatibility.
- Benchmark and calibration workflows now make quality claims measurable, auditable, and replayable.
- Topology context now reaches beyond static service maps into Terraform state and Kubernetes live-state imports.
- CODEOWNERS and uploaded ownership maps can now help route files and resources to accountable owners.
- UI design planning assets define the next dashboard/report review direction without changing runtime behavior.

## Highlights

### Integration and Automation

- Added a stable v1 report contract for API consumers.
- Added project-aware CLI advisory output for automation and CI use.
- Hardened the GitHub Action split-repository contract.
- Added PR comment formatting for advisory results.
- Added rerun-on-commit report comparison support so GitHub workflows can show what changed between advisory runs.
- Added a future adapter output contract for downstream workflow integrations.

### Benchmarks and Calibration

- Added a public benchmark corpus for measuring advisory quality.
- Added a benchmark runner that checks local analysis behavior against expected scenario outcomes.
- Added honest failure reporting so benchmark gaps are visible instead of overstated.
- Added outcome calibration metrics backed by persisted reviewer and deployment outcome history.
- Added incident backtesting to replay historical incident signals against current analysis behavior.
- Added risk trend review/export support for persisted calibration history.

### Topology and Context

- Made blast-radius topology context project-scoped and more trustworthy.
- Added a Terraform state topology connector.
- Added a Kubernetes live-state connector and related context freshness behavior.
- Improved topology-backed analysis, context completeness, and blast-radius validation.

### Ownership Mapping

- Added CODEOWNERS ingestion and ownership mapping for uploaded artifacts.
- Added ownership signals, escalation hints, and unmapped ownership subjects to report context.
- Preserved trusted repository-relative artifact paths across API, CLI, and UI upload flows.
- Hardened handling of unsafe or external artifact paths so internal sentinel markers are not exposed as trusted ownership.
- Supported duplicate basenames across trusted artifact paths while still rejecting exact duplicate trusted paths.

### UI and Design Direction

- Added new dashboard/report redesign planning assets for the next release cycle.
- Added local design asset validation around fonts and UI direction.
