# DeployWhisper v1.3.0

DeployWhisper v1.3.0 is the context and scanner-integration release. Compared
with v1.2.0, which established the React web UI as the primary application
surface, v1.3.0 deepens the advisory engine with context-source freshness,
scanner ingestion, external-evidence labeling, and explicit scanner conflict
handling through the UI, API, CLI, and share/PR outputs.

## Highlights

- Context sources now carry freshness, confidence, scope, conflicts,
  limitations, and evidence provenance through report, API, CLI, and React UI
  output.
- SARIF 2.1.0 and Semgrep JSON scanner findings can be imported as
  project/workspace-scoped external evidence.
- Reports distinguish DeployWhisper deterministic evidence from external scanner
  context so scanner severity does not silently become DeployWhisper severity.
- Scanner-vs-deterministic disagreements are surfaced as explicit conflicts with
  source details, freshness, recommended verification, uncertainty, and
  confidence impact.
- The application version now reports `1.3.0` from runtime API metadata and
  package metadata.

## Context Freshness and Provenance

- Added a context-source freshness ledger for parser, artifact, topology,
  incident, ownership, and evidence-derived context.
- Added evidence-to-context references so reviewers can connect evidence rows to
  the context source that shaped the report.
- Added UI rendering for context-source freshness, confidence, scope, conflicts,
  limitations, and evidence provenance in the React report screen.
- Added reload-time normalization for persisted context-source metadata,
  including stale incident handling, empty incident snapshots, malformed legacy
  rows, duplicate source identities, and forward-compatible additive fields.
- Added shared-report redaction support for context-source ledger fields and
  nested evidence provenance.

## Scanner Import and External Evidence

- Added scanner import persistence and repository support for normalized scanner
  findings.
- Added the scanner import API route and generated frontend OpenAPI artifacts.
- Added SARIF 2.1.0 ingestion with validation for version, messages, rules,
  severities, locations, URI safety, fingerprints, request limits, and
  project/workspace scoping.
- Added Semgrep JSON ingestion with validation for result shape, severities,
  paths, metadata, fingerprints, engine metadata, scan errors, and result-count
  limits.
- Added scanner import documentation covering supported formats, scope fields,
  duplicate/update behavior, validation errors, and report-safe metadata.
- Added brownfield database schema checks and migrations for context-source and
  scanner import persistence.

## Scanner-Aware Reports

- Added external-evidence labels to report detail payloads, React finding rows,
  evidence registers, share summaries, Markdown output, and plain-text output.
- Preserved Evidence Law behavior: scanner-only severe findings are external
  context and do not automatically become high or critical DeployWhisper
  findings.
- Improved mixed-context labeling so findings supported by both DeployWhisper and
  scanner evidence are labeled as including external context instead of being
  described as purely external.
- Updated report schema documentation and examples for external evidence labels,
  scanner context, and context-source provenance.

## Scanner Conflict Handling

- Added scanner conflict synthesis for disagreements between scanner findings and
  deterministic evidence or context freshness.
- Conflict output includes scanner source, deterministic evidence source,
  freshness for each side, recommended verification, uncertainty, and confidence
  impact.
- Added conflict serialization in API detail payloads, CLI JSON output, adapter
  output contracts, share summaries, compact PR/comment summaries, and the React
  report UI.
- Added safe Markdown and redacted-summary handling so scanner-controlled text
  does not leak raw HTML or sensitive source references in detail-omitted share
  output.
- Fixed protected shared-report unlocks so comparison URLs preserve
  `compare=previous`, and optional comparison lookup failures do not block a
  valid password unlock.

## Upgrade Notes

- Apply migrations before running against an existing database. This release adds
  context-source and scanner-import persistence migrations.
- Scanner imports remain advisory external evidence. They enrich DeployWhisper
  reports but do not replace deterministic scoring or Evidence Law.
- Shared reports are still intended for controlled sharing. Continue using
  password protection and filename redaction for sensitive reports.
